<Section icon={Shield} title="Channel Etiquette, Roles, and Posting Standards">

Good Discord etiquette helps your messages get seen and answered faster. The strongest posts are short, specific, and easy to reproduce. That matters even more in a project built around many custom classes, specs, and systems.

<Callout type="success" title="Better Posts Get Better Replies">
A clear report with steps, screenshots, and exact wording usually gets a faster answer than a vague complaint. Keep one issue per message when possible.
</Callout>

| Channel Pattern | Do This | Avoid This |
| --- | --- | --- |
| Announcements | Read, react, stay quiet | Asking support questions there |
| General Chat | Share builds, groups, events | Spamming off-topic noise |
| Bug Reports | One issue, full reproduction steps | Posting only “broken” |
| Class Feedback | Explain gameplay impact | Demanding balance changes without context |
| LFG or Guild Chat | State realm, time, goal | Posting unrelated ads |

<Section icon={Trophy} title="Support, Bug Reports, and Launch-Day Prep">

The best way to use the server is to separate three different needs: technical support, gameplay bugs, and feedback. Those are similar, but they do not belong in the same message. Organizing them correctly helps the team route your post faster.

| Problem | Best Channel Type | What To Include |
| --- | --- | --- |
| Login or launcher issue | Support | Error text, platform, time, screenshot |
| Gameplay bug | Bug reports | Repro steps, zone, spec, expected result |
| Balance concern | Class feedback | Build, gear, encounter, why it feels off |
| Guild recruitment | Community chat | Region, schedule, raid goal, contact |

Before a launch or major update, use a simple prep routine so you are not scrambling once the server gets busy.

<Steps>
  <Step num="1" title="Update everything early">
    Make sure the launcher, client, and any required files are already current before peak traffic hits.
  </Step>
  <Step num="2" title="Mute low-value noise">
    Silence channels you do not need so important notices stand out immediately.
  </Step>
  <Step num="3" title="Prepare report materials">
    Save screenshots, notes, and reproduction steps in one folder for quick posting.
  </Step>
  <Step num="4" title="Join the right channels early">
    Enter support, announcements, and class feedback channels before the rush starts.
  </Step>
</Steps>

<Callout type="info" title="Support Workflow">
If your issue needs a reply from staff, keep your post concise and factual. Save opinions for the feedback channel and keep support posts focused on the problem itself.
</Callout>

| Launch Prep Item | Status Goal | Benefit |
| --- | --- | --- |
| Updated launcher | Ready before peak time | Fewer setup delays |
| Notification settings | Important channels enabled | Faster alert response |
| Bug report notes | Saved in advance | Cleaner posts under pressure |
| Role selections | Already chosen | Less channel hunting |
